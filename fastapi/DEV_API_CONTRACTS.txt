Whitepapper Dev API Contracts
Date: 2026-03-24

Base path: /api/v1
Auth for public dev bundle endpoints: x-api-key header

1) GET /dev/project
- Headers:
  - x-api-key: <raw api key>
- Behavior:
  - Validates API key by hashed lookup.
  - Loads API key's project and requires project.isPublic == true.
  - Returns only public collections of the project.
  - Returns only published standalone papers in the project
    (projectId matches, collectionId is empty/null).
- Response:
  {
    "project": ProjectDoc,
    "collections": CollectionDoc[],
    "papers": PaperDoc[]
  }

2) GET /dev/collection?id=<collectionId>
   GET /dev/collection?slug=<collectionSlug>
- Headers:
  - x-api-key: <raw api key>
- Query:
  - Provide exactly one: id or slug
- Behavior:
  - Collection must belong to the API key project.
  - Collection must be public (isPublic == true).
  - Returns published papers for that collection.
- Response:
  {
    "collection": CollectionDoc,
    "papers": PaperDoc[]
  }

3) GET /dev/paper?id=<paperId>
   GET /dev/paper?slug=<paperSlug>
- Headers:
  - x-api-key: <raw api key>
- Query:
  - Provide exactly one: id or slug
- Behavior:
  - Paper must belong to the API key project.
  - Paper must be published (status == "published").
  - Does not block paper fetch by collection visibility.
  - Returns matching project/collection documents when those entities are public.
- Response:
  {
    "paper": PaperDoc,
    "project": ProjectDoc | null,
    "collection": CollectionDoc | null
  }

4) GET /projects/{project_id}/api-key
- Auth:
  - Bearer user token (owner only)
- Behavior:
  - Returns the project's API key metadata (no keyHash), or null if absent.
- Response:
  {
    "keyId": string,
    "ownerId": string,
    "projectId": string,
    "usage": number,
    "limitPerMonth": number,
    "isActive": boolean,
    "createdAt": string
  } | null

5) POST /projects/{project_id}/api-key
- Auth:
  - Bearer user token (owner only)
- Body:
  - none
- Behavior:
  - Creates one key per project (conflict if one already exists).
  - Returns key metadata + one-time rawKey.
- Response:
  {
    "keyId": string,
    "ownerId": string,
    "projectId": string,
    "usage": number,
    "limitPerMonth": number,
    "isActive": boolean,
    "createdAt": string,
    "rawKey": string
  }

6) PATCH /api-keys/{key_id}
- Auth:
  - Bearer user token (owner only)
- Body:
  {
    "isActive": boolean
  }
- Response:
  {
    "keyId": string,
    "ownerId": string,
    "projectId": string,
    "usage": number,
    "limitPerMonth": number,
    "isActive": boolean,
    "createdAt": string
  }

7) DELETE /api-keys/{key_id}
- Auth:
  - Bearer user token (owner only)
- Response:
  {
    "ok": true
  }
