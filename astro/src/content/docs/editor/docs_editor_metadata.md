Paper metadata controls how public paper pages appear in search and link previews.

## What metadata includes

Paper metadata covers:

- Basic SEO fields (`title`, `metaDescription`, `canonical`, `robots`, `keywords`)
- Open Graph fields
- Twitter fields
- JSON-LD fields

See [Paper metadata](/docs/seo/paper-metadata) for field mapping.

## Open metadata editor

In the editor top bar action menu, choose **Metadata**.

## Generate metadata

Use **Generate** to auto-fill metadata from title/body context. Generating replaces current draft values in the metadata dialog.

## Edit metadata manually

You can edit every metadata field, including image fields.

Typical manual overrides:

- `canonical` for syndication source control
- `robots` for indexing behavior
- `ogImage` and `twitterImage` for social preview quality

## Save behavior

Metadata dialog **Save** writes to the paper's in-editor state. Changes are persisted to backend when you save the paper.

## When metadata takes effect

Metadata affects public paper pages once the paper is publicly accessible and saved.
